一些疑问解答

为什么用one-hot编码词向量不好？

答：搜索eat food得到eat meat但是food与meat是one-hot编码会相互垂直，不存在相似性，所以很难搜索出来。解决方法：使用邻居的值来估计

为什么在训练时每个词使用两个向量来表示？

答：在实际的工程上效果更加好。更加容易收敛

在实际的应用上是使用context vector还是 center word vector？

答：使用center word vector

滑动窗口的词要考虑到位置关系吗？

答：不需要，实际效果会更好

为什么度量向量的相似性使用u*v而不用再除以他们的长度？

答：计算量复杂，实际效果相差不大。

我们注意到我们的目标函数需要计算语料里的所有词（T个）有可能是十亿量级，怎么解决？

答：SGD。用一个词来更新所有的词向量

在skip gram 中的 softmax中的V是指窗口的V还是全部语料的V

答：是全部语料的V，也就是说V是十亿量级的级别

那V这么大要怎么解决？

答：在skip gram中使用negative sampling

目标函数的凸函数？

答：不是，为什么？

向量的内积作为相似性度量，我们可以使用欧几里得距离吗？

答：内积效果会更加好

常用的word2vec模型

使用Skip gram

* V：字典中词数
* d：每个词的维度
* one-hot的形式输入
* 在d x V的大center word matrix中找出对应的v_c
* d x 1的v_c 跟第一个V x d context word matrix相乘（t - 3位置的权重，假设窗口m = 3）
* d x 1的v_c 跟第二个V x d context word matrix相乘（t - 2位置的权重）
* …
* 最后得到 V x 1的向量，使用softmax来归一化我们的数值
* 把真实的label输入到目标函数，来计算我们的loss

使用negative sampling

符号说明：

* 使用sigmoid做数值压缩
* T超参数，选T个windows做一次更新
* 根据P(w) 这个概率挑选k个不是共同出现（co-occur）的词作为负采样
* Z是什么？以后补充？
* 3/4次方是为了惩罚概率大的词，使得不太容易被抽到，稀罕的词容易被抽到（自己画个根号的函数就知道了）
* J(theta)是目标函数，前一项两个向量越相似越等于0，后一项两个向量越不相似越不等于0，所以我们maximize J(theta)就会达到目的。
